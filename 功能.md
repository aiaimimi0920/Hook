# Hook 功能测试清单 (Feature Manifest)

这份文档记录了当前 Hook 项目中已实现的所有核心交互与功能，可用作回归测试（Regression Testing）的参照表。

## 🎮 基础操作 (Basic Operations)

### 1. 截图与采集
- **[Ctrl + 1] 截图 (Capture)**
  - **操作**: 按下快捷键，屏幕出现选框。
  - **预期**: 选定区域后，Canvas 上应立即生成一个新的图片节点。

### 2. 节点管理
- **[Shift + 1] 添加节点 (Add Node)**
  - **操作**: 选中某节点或鼠标悬停某处，按下快捷键。
  - **预期**: 弹出 Actions Menu（操作菜单），点击菜单项可创建新节点并自动连接。
- **[Delete / Backspace] 删除节点**
  - **操作**: 选中节点后按键。
  - **预期**: 节点被移除，相关连线自动断开。
- **[Ctrl + C / Ctrl + V] 复制粘贴**
  - **操作**: 选中节点复制，移动鼠标后粘贴。
  - **预期**: 在鼠标当前位置生成节点副本，参数与原节点一致。

## 🖱️ 节点交互 (Node Interaction)

### 3. 移动与布局
- **自由拖拽**
  - **操作**: 左键按住节点标题栏或内容区拖动。
  - **预期**: 节点随鼠标移动，连线实时跟随。
- **[Alt + 拖拽] 智能吸附 (Snap Align)**
  - **操作**: 按住 Alt 键拖动节点。
  - **预期**: 节点靠近其他节点边缘时会自动吸附对齐（X轴或Y轴）。
- **[Ctrl + 拖拽] 层叠推挤 (Cascade)**
  - **操作**: 按住 Ctrl 键拖动节点到其他节点上方。
  - **预期**: 节点会自动堆叠或以层叠形式排列，Z轴顺序正确。

### 4. 尺寸与视图
- **[Ctrl + 滚轮] 缩放节点**
  - **操作**: 鼠标悬停节点，按住 Ctrl 滚动。
  - **预期**: 节点尺寸以中心为锚点缩放。**重点测试**: 缩放时连线端点应紧贴节点边缘，不应错位。
- **[Alt + 滚轮] 调整透明度**
  - **操作**: 鼠标悬停节点，按住 Alt 滚动。
  - **预期**: 节点不透明度变化 (0.1 ~ 1.0)。
- **[双击] 最小化/还原**
  - **操作**: 左键双击节点。
  - **预期**: 节点在“完整视图”与“裁剪视图(Minified)”间切换。
- **[点击按钮] 适配外框 (Fit Frame)**
  - **操作**: 点击节点下方工具栏的“合拢外框”按钮。
  - **预期**: 节点尺寸自动裁剪，仅保留图片宽高比区域，不拉伸图片，连线位置自动更新。

### 5. 文件交互
- **[Shift + 拖拽] 导出文件**
  - **操作**: 按住 Shift 键，将节点图片拖拽到 Windows 文件夹或桌面。
  - **预期**: 鼠标显示“复制”图标，松开后在目标文件夹生成 `.png` 文件。

## 🛠️ 高级功能 (Advanced)

### 6. 参数与界面
- **[Tab] 切换参数面板**
  - **操作**: 选中节点按 Tab。
  - **预期**: 节点右侧/下方展开参数面板。
- **[Ctrl + 4] 纯净模式 (Clean View)**
  - **操作**: 按快捷键。
  - **预期**: 隐藏所有连线、端口插槽，仅显示图片节点内容。再次按下恢复。

### 7. AI 辅助 (未完全覆盖测试)
- **[Alt + 2] OCR 开关**
  - **操作**: 选中节点按 Alt+2。
  - **预期**: (如已识别) 显示/隐藏文字识别覆盖层。
- **[Alt + 3] 翻译开关**
  - **操作**: 选中节点按 Alt+3。
  - **预期**: 显示/隐藏翻译结果。

## ⚠️ 待验证/未提及的测试点 (Gap Analysis)

以下功能存在于代码库中，但在近期的重构对话中未被重点测试，建议后续关注：

1. **Shader 实时渲染性能**:
   - 拖动 Shader 节点参数滑块时，渲染是否流畅？是否存在卡顿导致 UI 无响应？
2. **多节点操作 (Multi-Select)**:
   - 目前仅测试了单节点操作。框选多个节点后，`Delete` 是否能批量删除？`Shift+Drag` 是否能批量导出？
3. **数据持久化 (Session Save/Load)**:
   - 重启软件后，节点的`最小化`状态、`透明度`设定、以及`Fit Frame`后的尺寸是否完美恢复？
4. **环路连接 (Loop Detection)**:
   - 如果将节点的输出连回自己的输入（或 A->B->A），系统是否会死循环或崩溃？
5. **极值测试**:
   - 将节点缩放到极小 (20px) 或极大 (2000px) 时，交互是否依然正常？
